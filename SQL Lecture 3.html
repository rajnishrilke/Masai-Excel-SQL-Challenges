Deleting Data in SQL
Delete statement is used to remove one or more rows from the table
Indicate table_name in DELETE statement from which you want to remove the rows
Specify the condition in WHERE for rows which needs to be deleted.
If you want to delete all the rows, omit the WHERE condition here
Syntax:
DELETE table_name WHERE condition;
SQL
In general, the DELETE statement consists of the following parts:
Keyword: DELETE
Name of the table whose rows are to be deleted.
Keyword: WHERE
Condition to apply in the WHERE Clause to filter on the exact rows which need to be deleted.
Unlike updating single fields at a time in UPDATE statement, the DELETE statement deletes the entire row from the table.
Reading the table:
Syntax:
SELECT * FROM hacker;

Syntax:

Filtering Data using WHERE Clause in SQL
The WHERE clause is used to filter records.
It is used to extract only those records that fulfill a specified condition.
General Syntax:
SELECT * FROM table_name WHERE CONDITION
SQL
In general the WHERE clause is applied on columns of tables which may or may not have been included in the SELECT statement but are a part of the tables from which the data is being selected.
Only those records will be extracted who are satisfying the specified condition in WHERE clause.
It can be used with SELECT, UPDATE, DELETE statements.
WHERE Clause implements in row operations
WHERE Clause is used with single row function like UPPER, LOWER etc.
Applying mathematical operators to filter using the WHERE Clause:
The “EQUAL To” Operator
The EQUAL To operator is a mathematical operator which is used to equate row values of a specified column with a specific integer value.
Syntax:
select * from student_marks WHERE maths = 20;
SQL
Output:
From the above output, we see that the WHERE condition we had applied filtered out all the rows and printed for us in the output wherever the marks in Math's were equal to “20” just as we had specified in the query.
The GREATER THAN Operator
Let’s say we want to filter out records of students who score above 50 in science, we write the following query:
Syntax:
select * from student_marks WHERE science > 90;
SQL
Output:
From the above output, we see that similar to the WHERE condition we had applied to filter out rows which had Math marks equal to 20, in this case, we have filtered rows having details of those candidates whose marks in science are greater than 90.
The GREATER THAN or EQUAL TO Operator
We has so far seen the use of the two different mathematical comparison operators in different scenarios, but lets see if we can also use them in combination. That is, can we filter on rows which are equal to and also greater than a specified value for a certain column:
Syntax:
select * from student_marks WHERE english >= 97;
SQL
Output:
Here we can see that we get rows which have values in the English column both equal to and greater than 97, as specified in the query.
Similar to the greater than operator, we can also apply the less than operator both individually and in combination with the equal to operator to filter records according to our needs using the WHERE clause.
Filtering with a combination of conditions in the WHERE Clause
SQL contains many logical operators which can be used to combine different conditions on which you want to filter your data.
As an example, we can use the “AND” operator to filter on two conditions which are to definitely followed by the Server to return the data. The “OR” operator on the other hand gives SQL server the liberty to filter the data on either of the specified 2 conditions.
Case 1: AND Operator

Case 3: BETWEEN Operator
What do you do when you want your output to have values in a particular range of values?
You code the following:
Syntax:
select * from student_marks WHERE maths >= 40 and maths <= 45;
SQL
Output:
The BETWEEN operator does exactly the same as shown in the above query. It is much more readable and much more syntactically appropriate because when you are writing long queries, it becomes very easy to read.
One thing to note for the BETWEEN operator is both the starting and ending values are included - this is one of the things you need to remember when using BETWEEN.
Syntax:
select * from student_marks WHERE maths BETWEEN 50 and 55;

COUNT() Function in SQL
So far we have applied varied functions and gotten the data as it is, we have not really made any new changes of our own to the output, we have gotten the same row values of the tables but with few filters.
Many a times you will need to count the number of rows. Lets say you want to count how many products are there in a particular category or you want to count the number of students in a class. For such cases we use the function called COUNT().
The COUNT() function can be used in many different ways - as an aggregate function - as an aggregate window function, we shall be getting to know about more in the further classes.
The function can be applied on any of the columns.
No matter the column name on which you apply the count function to , it gives the same output - which is the number of rows.
Examples:
Syntax 1:
select COUNT(id) from student_marks;
SQL
Output 1:
Syntax 2:
select COUNT(gender) from student_marks;
SQL
Output 2:
Syntax 3:
select COUNT(name) from student_marks;
SQL
Output 3:
So, it does not matter which column you putting in the COUNT function, it is a general understanding/convention which makes more sense to mention the name of the column which you want to count for a particular problem statement, so when you come back after an year and read the query, you are able to read the same query and understand why you wanted to count on that particular column you had mentioned in the query.
COUNT Function can be combined with any of the where conditions. It will then only count those rows which satisfy that WHERE condition.
Syntax:
select COUNT(id) from student_marks WHERE maths = 20 and english > 50;
